## 项目主要需求

### 项目概述

【全栈Web应用】实验室设备管理系统。面向高校和科研院所的实验室，支持设备资产登记、借用归还、维护保养、使用预约和设备状态看板。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **设备（Equipment）** — 字段：设备名称、设备编号、分类ID、品牌型号、序列号、购买日期、购买价格、存放位置（楼栋-房间-柜号）、资产状态（Available/InUse/Maintenance/Retired/Lost）、责任人ID、供应商、保修到期日、设备图片URL
   - 关联：1台设备 → 多条借用记录、多条维护记录、多条预约记录
   - 业务动作：登记入库、报废、转移责任人
   - **贯穿全栈**

2. **借用记录（BorrowRecord）** — 字段：设备ID、借用人ID、借用日期、预计归还日期、实际归还日期、借用事由、审批状态（Pending/Approved/Rejected/Returned/Overdue）、审批人ID、归还状况（Good/Damaged/Lost）
   - 业务动作：提交借用、审批、确认归还
   - **贯穿全栈**

3. **维护记录（MaintenanceRecord）** — 字段：设备ID、维护类型（Preventive/Corrective/Calibration/Cleaning）、维护内容、维护日期、下次维护日期、维护费用、维护人ID、维护结果（Pass/Fail/NeedsFollowUp）
   - 业务动作：创建维护计划、执行维护、记录结果
   - **贯穿全栈**

4. **预约记录（Reservation）** — 字段：设备ID、预约人ID、预约开始时间、预约结束时间、使用目的、审批状态（Pending/Approved/Rejected/Cancelled）、审批人ID
   - 业务动作：创建预约、审批预约、取消预约
   - **贯穿全栈**

5. **设备分类（EquipmentCategory）** — 字段：分类名称、父分类ID、描述、图标
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **设备总览仪表盘** `/dashboard` — 消费：Equipment + BorrowRecord + MaintenanceRecord
   - 设备状态分布（饼图）、本月借用次数TOP10、即将过保设备列表、待审批借用/预约数量
   - 使用 `<StatusBadge>`, `<StatCard>`, `<AlertBanner>`

2. **设备管理** `/equipment` — 消费：Equipment + EquipmentCategory + BorrowRecord
   - 设备列表（树形分类+搜索），设备详情（基本信息+借用历史+维护记录+预约日历），新增/编辑设备
   - 使用 `<StatusBadge>`, `<CalendarCell>`

3. **借用管理** `/borrow` — 消费：BorrowRecord + Equipment
   - 借用申请列表（按状态Tab），提交借用（选设备+填信息），审批借用，确认归还
   - 使用 `<StatusBadge>`, `<StepIndicator>`

4. **预约管理** `/reservations` — 消费：Reservation + Equipment
   - 设备预约日历视图（按设备切换），创建预约（选时间段），审批预约
   - 使用 `<CalendarCell>`, `<StatusBadge>`

5. **维护管理** `/maintenance` — 消费：MaintenanceRecord + Equipment
   - 维护计划日历（即将到期高亮），执行维护记录，维护费用统计
   - 使用 `<StatusBadge>`, `<CalendarCell>`, `<StatCard>`

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / LabManager / Researcher / Student
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮显隐（Student不能审批）+ 类型定义

2. **操作日志**
   - 记录设备登记、借用审批、归还确认、维护记录等关键操作
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 前端日志页 + 类型定义

3. **全局异常处理**
   - 触达文件层：后端异常过滤器 + controller业务异常 + 前端ErrorBoundary + API拦截器 + 错误码枚举

### 共享枚举/常量（≥ 2 组）

1. **资产状态 AssetStatus**：Available / InUse / Maintenance / Retired / Lost
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`

2. **借用审批状态 BorrowStatus**：Pending / Approved / Rejected / Returned / Overdue

3. **维护类型 MaintenanceType**：Preventive / Corrective / Calibration / Cleaning

4. **归还状况 ReturnCondition**：Good / Damaged / Lost

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 所有页面
2. **`<CalendarCell>`** — 设备详情、预约管理、维护管理
3. **`<StatCard>`** — 仪表盘、维护管理
4. **`<StepIndicator>`** — 借用管理（申请→审批→使用→归还）
5. **`<AlertBanner>`** — 仪表盘（过保预警、逾期借用）

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useBorrowFlow`** — 封装借用全流程逻辑（申请→审批→使用→归还）
2. **`usePagination`** — 通用分页
3. **`formatCurrency`** — 金额格式化（设备价格、维护费用）

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色校验
3. **`auditLogMiddleware`** — 操作日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`lab-equipment`
- **前端端口**：18801 / 80
- **后端端口**：19301 / 3000
- **前端**：React 18 + TypeScript + Vite
- **UI 库**：Ant Design 5
- **图表**：ECharts
- **后端**：NestJS + TypeScript
- **数据库**：MySQL 8.0
- **ORM**：TypeORM
- **认证**：JWT
- **状态管理**：Zustand
- **构建工具**：Vite

### 部署要求

Docker Compose 一键部署。
- 顶层 `name: lab-equipment`
- 不写 `version:` 字段
- MySQL 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

### Git 要求

`git init` + `.gitignore`。
